Что требуется сделать для работы проекта

УСТАНОВКА

Если Вы работаете на Linux, должно быть достаточно запустить скрипт install.sh:
$ ./install.sh
(Работает на Ubuntu, если не работает, см. инструкции ниже)

В папке editor-master/tree есть программа buildAST.py. 
Следует проверить значение переменной PYTHON_LAUNCH (начало файла, около 9-10 строк). 
По умолчанию присваивается значение в зависимости от операционной системы
(на Windows это обычно просто "python", на Ubuntu "python3"), если оно присвоено неверно,
надо поправить (это имя, по которому запускается Python из командной строки).

Если работаете на Windows, требуется поставить ANTLR версии 4.7.2 вручную.
Это сложнее и займёт чуть больше времени. 
Для этого надо по ссылке https://www.antlr.org/download/index.html 
найти версию 4.7.2 и скачать JAR-файл.
Инструкцию по установке можно найти тут: https://stackoverflow.com/questions/41021963/how-to-install-antlr4
Идея в том, чтобы добавить папку с JAR файлом в PATH + в этой папке
должны лежать *.bat-файлы, которые запускают сам JAR файл с помощью JAVA.
Батники лежат в папке "win_setup".

Необходим ANTLR версии 4.7.2, номер версии крайне важен.
Должно быть соответствие между версией ANTLR,
установленной в системе для работы из командной строки, и версией ANTLR,
которая является библиотекой на Python3
(т. к. из командной строки запускается код, обрабатывающий грамматику *.g4,
после чего библиотека работает с артефактами).

Требуется, чтобы был установлен Graphviz: https://graphviz.org/download/

Ниже (до секции ЗАПУСК) инструкция для Linux (как установить всё вручуню, не
используя install.sh)

Установка ANTLR для командной строки:
$ sudo apt-get antlr4
Проверка, что установилась нужная версия: надо в консоли ввести antlr4,
должен быть номер версии 4.7.2.
Установка ANTLR нужной версии для Python3 должна произойти автоматом после запуска:
$ pip install requirements.txt -r
(В requirements.txt должна быть указана версия 4.7.2)


ciaoVisitor.py нельзя изменять/генерировать заново,
так как он модифицирован (туда добалвен метод get_graph).

Для открытия *.svg картинок в Linux можно использовать команду
$ display <image_name>.svg
Для этого можно установить ImageMagic:
$ sudo apt-get install imagemagick
*.svg картинки можно открывать в браузере

Перед первым запуском надо установить зависимости npm в директории front.
$ cd front

Далее:
$ npm install


ЗАПУСК

Запуск приложения происходит в несколько этапов.
1. в директории editor-master запустить main.py:
$ python3 main.py
Или (для Windows)
$ python main.py

2. не закрывать запущенное приложение, в другом терминале открыть директорию front
и в ней выполнить:
$ yarn start
На некоторых машинах это выдаст ошибку, но можно запустить с помощью:
$ npm start
Картинки/код для пользователя должны появляться в папке front/public.
